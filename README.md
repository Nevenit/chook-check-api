# Chook Check API

Community price-data backend for the [Chook Check](https://github.com/Nevenit/Chook-Check) browser extension. Cloudflare Worker + D1.

## License

**AGPL-3.0**. If you run a modified version of this service, you must publish your source. Full text in [LICENSE](LICENSE).

## What it does

- Accepts batched price observations from extension users who have opted into contribution
- Returns community-aggregated stats per product, gated by a 3-contributor crowd quorum
- Returns trending price movements over configurable periods
- Lets contributors permanently delete all their data with one DELETE request

It deliberately does **not**:

- Log IP addresses against observations (IPs are used transiently for rate limiting and never persisted)
- Track individuals across submissions (the contributor ID is the only identifier, generated by the extension on install as a random UUID)
- Use any third-party analytics or telemetry

## Architecture

- [Hono](https://hono.dev) on Cloudflare Workers
- [Cloudflare D1](https://developers.cloudflare.com/d1/) (SQLite at the edge) for storage
- [Zod](https://zod.dev) for input validation
- Vitest with `@cloudflare/vitest-pool-workers` for tests

## Endpoints

| Method | Path | Purpose |
|---|---|---|
| `POST` | `/api/observations` | Batch submit (1–50 per request, 60/hr per contributor) |
| `GET` | `/api/products/:productId/stats` | Per-product aggregates with crowd quorum |
| `GET` | `/api/products/search` | Search by name or brand |
| `GET` | `/api/trends` | Biggest period-over-period changes |
| `DELETE` | `/api/contributor/:contributorId` | Delete all data for a contributor |
| `GET` | `/health` | Health check |

Schemas in `src/lib/schemas.ts`. Response shapes in `src/lib/types.ts`.

## Local development

```bash
npm install
npm run dev              # wrangler dev — local miniflare D1
npm test                 # one-shot integration tests
npm run test:watch
```

Tests run against an in-memory D1 instance, exercising the real query layer.

## Deployment

```bash
npm run deploy           # wrangler deploy
```

D1 binding is configured in `wrangler.toml`; production database is `chook-check`.

### Migrations

```bash
wrangler d1 migrations create chook-check <name>
wrangler d1 migrations apply chook-check          # local
wrangler d1 migrations apply chook-check --remote # production
```

Never edit a previously-applied migration. Add a new one.

## Schema overview

```sql
CREATE TABLE observations (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  product_id TEXT NOT NULL,
  product_name TEXT NOT NULL,
  brand TEXT, category TEXT, gtin TEXT,
  store_chain TEXT NOT NULL,
  price_cents INTEGER NOT NULL,
  was_price_cents INTEGER, unit_price_cents INTEGER, unit_measure TEXT,
  promo_type TEXT,
  is_personalised INTEGER NOT NULL DEFAULT 0,
  contributor_id TEXT NOT NULL,
  browser TEXT, state TEXT, city TEXT, store_name TEXT,
  observed_at TEXT NOT NULL,
  submitted_at TEXT NOT NULL
);
```

Plus a `rate_limits` table keyed by (contributor ID or IP, endpoint).

No `pageUrl`, no client IP, no email — by design.
